### Assembler planning
- How to create the MD byte.
Put each parameter in a vector
Create a new map. `std::map<std::string, uint8_t> MD_DETERMINE = {{"imm", 0b00}, {"reg", 0b01}, {"addr", 0b10}};`
Create a variable. `std::vector<std::string> MD_final = {};`
Iterate through each item of the vector
If the item starts with R, then it's a register
If the item starts with '@', then it's an address
If the item starts with neither, it's an immediate
Use MD_DETERMINE.at() on the item
With the response, append it to MD_final.
+ Note that instructions with an MD byte follow C, B, A - A, B, C.
+ C, B, A is the MD byte, and A, B, C are the parameters.
After all items have been iterated through, call output.push_back() with "0b" prefix before MD_final. Still use the 8-bit padding to be consistent.

### VM/VCPU planning
- How GDI will be implemented in the "kernel" (INT)
+ Note that there is no 'byte input limit' parameter given; This is intentional.
INT is received
+ R0 is the input source (user's keyboard, or mouse in the future)
+ R1 is the address to begin writing to
interrupt() looks at R2 and goes to the address in it
Interrupt() calls `std::getline(std::cin >> std::ws, X);` (X being a placeholder for now) to get the user input
When the user finally presses enter, Interrupt() writes that input to the address given in R2
Interrupt sets R0 to the amount of bytes the user wrote.
Interrupt() returns the control back to the program.

- How endianness will be handled when it's finally time

But eventually, we will transition to Little Endian to make some math operations faster.
